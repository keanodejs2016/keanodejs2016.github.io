11 Exercise: /Orders/

In this exercise you should finish the Store API by enabling it to handle orders.

You should now have 2 collections in your store database:

  • users
  • products

And you should be able to view an API in the browser and get, post, put and delete through this api.

On this url you can se an example of how an order should be structured.

An order has information about one or more products in an order. It has information about the user who placed the order, and it has information about the status of the order.

Your job is to create 5 routes for the orders. A get, a get one, a post, a put, and a delete route.
